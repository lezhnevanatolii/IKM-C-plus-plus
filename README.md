Отчёт по Итоговому Контрольному Мероприятию по предмету "Языки программирования C++".

Тема проекта: Разработка объектно-ориентированного приложения на языке C++ для автоматизации учёта заказов в интернет-магазине.

Цель работы: Создание приложения, реализующего функциональность системы управления заказами интернет-магазина, с использованием принципов объектно-ориентированного программирования и соблюдением требований Стэнфордского стиля кодирования.

Задачи проекта:
1. Проанализировать предметную область «Учёт заказов в интернет-магазине» и выделить ключевые сущности.
2.Разработать иерархию классов, соответствующую структуре реляционной базы данных проекта.
3.Реализовать модульную архитектуру приложения с разделением на заголовочные и исходные файлы.
4.Спроектировать интуитивно понятный консольный пользовательский интерфейс с многоуровневым меню.
5.Внедрить систему валидации и обработки пользовательского ввода для предотвращения ошибок.
6.Реализовать паттерн Singleton для управления единым хранилищем данных в памяти.
7.Обеспечить функциональность CRUD-операций для всех сущностей системы.
8.Реализовать систему отчётности для анализа данных.

АЛГОРИТМ РЕШЕНИЯ ЗАДАЧИ:

ЭТАП 1: АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ
Выявление сущностей: На основе базы данных «Учёт заказов в интернет-магазине» определены ключевые сущности: Клиент, Товар, Категория товара, Заказ, Позиция заказа, Адрес доставки, Сотрудник, Статус заказа.
Определение атрибутов: Для каждой сущности выделены характеристики (поля), соответствующие полям таблиц БД.
Установление связей: Определены отношения между сущностями (один-ко-многим, многие-ко-многим через ассоциативные классы).

ЭТАП 2: ПРОЕКТИРОВАНИЕ АРХИТЕКТУРЫ ПРИЛОЖЕНИЯ
Разработка диаграммы классов: Создана объектная модель, отображающая классы, их атрибуты, методы и взаимосвязи.
Выбор паттернов проектирования:
Singleton – для класса Database, обеспечивающего единую точку доступа к данным.
Модульный подход – каждый класс в отдельных файлах .h и .cpp.
Планирование модулей: Приложение разделено на логические модули:
Модули сущностей (Customer, Product, Order и др.)
Модуль управления данными (Database)
Модуль пользовательского интерфейса (UI, AppController)
Модуль утилит (InputUtils, StringUtils)

ЭТАП 3: РЕАЛИЗАЦИЯ КЛАССОВ СУЩНОСТЕЙ
Создание заголовочных файлов (.h): Объявление классов с полями и публичными методами (геттеры, сеттеры, методы отображения).
Создание исходных файлов (.cpp): Реализация методов классов.
Соблюдение инкапсуляции: Все поля классов объявлены как private, доступ через публичные методы.
Реализация бизнес-логики: Методы для операций с данными (например, reduceStock(), addItem()).

ЭТАП 4: РЕАЛИЗАЦИЯ СИСТЕМЫ ХРАНЕНИЯ ДАННЫХ
Класс Database: Реализация паттерна Singleton.
Хранение данных: Использование контейнеров std::vector для коллекций объектов.
Методы доступа: Реализация CRUD-операций (добавление, поиск, обновление).
Генерация идентификаторов: Автоматическая нумерация ID для новых объектов.

ЭТАП 5: РАЗРАБОТКА ПОЛЬЗОВАТЕЛЬСКОГО ИНТЕРФЕЙСА
Многоуровневое меню: Иерархическая структура:
Главное меню → Меню модулей → Операции
Класс UI: Статические методы для отображения меню и сообщений.
Класс AppController: Управление логикой навигации и обработка выбора пользователя.
Визуальное оформление: Использование разделителей, заголовков, форматированного вывода.

ЭТАП 6: РЕАЛИЗАЦИЯ СИСТЕМЫ ВАЛИДАЦИИ ВВОДА
Класс InputUtils: Статические методы для безопасного ввода разных типов данных.
Проверка диапазонов: Контроль числовых значений (минимальное/максимальное).
Валидация строк: Проверка длины, формата (email, телефон).
Обработка ошибок: Информативные сообщения при некорректном вводе с повторной попыткой.

ЭТАП 7: РЕАЛИЗАЦИЯ БИЗНЕС-ЛОГИКИ И ОТЧЁТНОСТИ
Создание заказов: Пошаговый процесс:
Выбор клиента → Добавление товаров → Расчет суммы → Фиксация
Обновление остатков: Автоматическое уменьшение количества товаров при создании заказа.
Система отчётов:
Статистика магазина
Товары с низким остатком
Продажи по категориям
Управление статусами: Изменение статуса заказа с валидацией переходов.

ЭТАП 8: ТЕСТИРОВАНИЕ И ОТЛАДКА
Модульное тестирование: Проверка отдельных классов и методов.
Интеграционное тестирование: Проверка взаимодействия модулей.
Функциональное тестирование: Проверка всех сценариев использования.


СТРУКТУРА КОДА ПРИЛОЖЕНИЯ:

Проект реализован как консольное объектно-ориентированное приложение на C++ с модульной архитектурой.
Код организован в три логических слоя:

1) СЛОЙ ДАННЫХ (Data Layer)
7 классов-сущностей, представляющих бизнес-объекты интернет-магазина. Каждый класс соответствует таблице из БД и содержит поля данных, геттеры/сеттеры и метод отображения.

2) СЛОЙ ЛОГИКИ (Business Logic Layer)

Database – паттерн Singleton для централизованного управления коллекциями объектов в оперативной памяти. Обеспечивает CRUD-операции, генерацию ID и поиск объектов.
AppController – оркестратор приложения, связывающий пользовательский интерфейс с бизнес-логикой.
Order – содержит сложную логику работы с заказами: добавление/удаление позиций, расчет сумм, управление статусами.

3) СЛОЙ ПРЕДСТАВЛЕНИЯ (Presentation Layer)

UI – отрисовка многоуровневого меню, заголовков, сообщений об ошибках и успехах.
InputUtils – полная система валидации ввода: проверка типов данных, диапазонов, форматов (email, телефон), длины строк.
StringUtils – вспомогательные операции обработки текста.

ПАТТЕРНЫ ПРОЕКТИРОВАНИЯ:

Singleton (Database) – единая точка доступа к данным
Модульность – разделение на .h/.cpp файлы
Инкапсуляция – все поля private с публичными методами доступа
Разделение ответственности – каждый класс решает одну задачу

ПОТОК ДАННЫХ:

Пользователь → InputUtils (валидация) → AppController → Database → Объекты
          ↓                             ↓
        UI (меню)                    UI (результаты)

КОМПИЛЯЦИЯ И ЗАВИСИМОСТИ:

Стандарт C++11
Заголовки: <iostream>, <string>, <vector>, <algorithm>, <ctime>

Сборка: g++ -std=c++11 -o online_store online_store_single.cpp

Загрузка тестовых данных: Автоматическое заполнение системы демонстрационными данными.
